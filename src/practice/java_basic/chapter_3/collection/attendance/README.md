### 실습 제목 : 출석부 관리 시스템 만들기
### 문제 설명
학생 이름을 저장하는 출석부를 만든다.
- ArrayList에 학생 이름을 저장한다.
- 같은 이름이 여러 번 들어갈 수 있다.
- HashSet으로 중복을 제거한다.
- 반복문으로 학생 명단을 출력한다.
- 특정 학생을 삭제한다.
- 삭제 후 다시 출력한다.

---
### 이 문제에서 다루는 것

| 종류            | 사용 |
| ------------- | -- |
| 변수            | O  |
| ArrayList     | O  |
| HashSet       | O  |
| add()         | O  |
| remove()      | O  |
| 반복문(for-each) | O  |
| 출력            | O  |
| 컬렉션 간 데이터 복사  | O  |

---
### 이런 문제 왜 필요?
출석부를 생각해보자
```
민수
철수
영희
민수
철수
```
사람이 여러 번 체크하면 이름이 중복될 수 있다.

하지만 실제 출석 인원은 중복을 제거한 결과가 필요하다.
```
민수
철수
영희
```
컬렉션은 이런 문제를 해결하기 위해 만들어졌다.

### 코드 작성 전 생각하는 순서
#### 1. 무엇을 저장할 것인가?
학생 이름 → String

#### 2. 여러 명을 저장해야 하는가?
ㅇㅇ → 컬렉션 필요함
#### 3. 순서가 필요하고 중복도 허용할 것인가?
ㅇㅇ → ArrayList 사용
#### 4. 중복 제거도 해보고 싶은가?
ㅇㅇ → HashSet 사용
#### 5. 반복문으로 출력할 것인가?
ㅇㅇ → for-each(향상된 for문) 사용
#### 6. 삭제 기능도 추가할 것인가?
ㅇㅇ → remove() 사용

### 왜 이 단계에서 이런 코드를 사용하는가?
#### 1단계
학생 여러 명을 저장해야 한다.
```
ArrayList<String>
```
#### 2단계
학생을 추가해야 한다.
```
add()
```
#### 3단계
학생 전체를 확인해야 한다.
```
for-each(향상된 for문)
```
#### 4단계
중복 제거를 해보고 싶다.
```
HashSet
```
#### 5단계
학생을 삭제하고 싶다.
```
remove()
```

### 핵심 원리
#### ArrayList
```
순서 O
중복 O
```
창고처럼 데이터를 차례대로 저장한다.

#### HashSet
```
순서 X
중복 X
```
중복 제거용 필터라고 샏각하면 된다.

#### 향상된 for문(for-each)
```
컬렉션 안의 데이터를 하나씩 꺼내는 반복문
```
#### remove()
```
원하는 데이터를 삭제
```

### 반드시 알고 넘어가야 하는 것.
## 1
```
ArrayList
```
순서를 유지하고 중복을 허용한다.
## 2
```
HashSet
```
중복을 허용하지 않는다.
## 3
```
add()
remove()
```
컬렉션에서 매우 자주 사용되는 기능이다.
## 4
for-each(향산된 for문)는 컬렉션을 출력할 때 가장 많이 사용하는 반복문이다.
