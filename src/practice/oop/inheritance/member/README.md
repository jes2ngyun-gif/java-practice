# 온라인 쇼핑 주문 할인 구조
## 회원 등급별 주문 금액 계산기
## 이 문제에서 이용하는 것

| 사용 개념      | 사용 이유                    |
| ---------- | ------------------------ |
| 클래스/객체     | 회원을 객체로 만들기 위해           |
| 생성자        | 회원 이름, 주문 금액 초기화         |
| private 필드 | 데이터를 직접 건드리지 못하게 보호      |
| getter     | 필요한 값만 안전하게 조회           |
| 상속 extends | 일반 회원과 VIP 회원의 공통 구조 재사용 |
| 오버라이딩      | VIP 회원은 할인 계산 방식을 다르게 적용 |
| if 조건문     | 주문 금액이 유효한지 검사           |
| 메서드 분리     | 계산, 출력 역할을 나누기 위해        |

---
## 문제 설명
온라인 쇼핑몰에는 일반 회원과 VIP 회원이 있다.

일반 회원은 주문 금액 그대로 결제하고,
VIP 회원은 주문 금액에서 10% 할인을 받는다.

공통 정보는 부모 클래스`Member`에 작성하고,
VIP만 다른 계산 방식은 자식 클래스 `VipMember` 에서 재정의한다.

---
## 코드 조건
1. `Member` 클래스 만들기
2.`VipMember` 클래스가 `Member`를 상속받기
3. 회원 이름과 주문 금액은 `private`으로 만들기
4. 주문 금액이 0보다 작으면 0으로 저장하기
5. 일반 회원은 원래 금액 그대로 결제
6. VIP 회원은 10% 할인된 금액 결제
7. `calculatePayment()` 메서드를 VIP에서 오버라이딩하기
8. `Main`에서 일반 회원과 VIP 회원 객체를 각각 만들어 실행하기

## 코드 작성 전에 생각하는 순서
```
1. 회원에게 공통으로 필요한 정보는 무엇인가?
   → 이름, 주문 금액

3. VIP 회원만 다르게 동작해야 하는 부분은 무엇인가?
   → 결제 금액 계산 방식

4. 그러면 부모 클래스에는 공통 구조를 두고,
   자식 클래스에서는 다른 계산 방식만 다시 작성하면 된다.
```

## 핵심 원리
상속은 **공콩 코드를 부모 클래스에 두고, 자식 클래스가 필요한 부분만 다르게 확장하는 구조**이다.

여기서 `Member` 는 공통 구조를 가진 부모 클래스이고,
`VipMember`는 일반 회원과 대부분 같지만 **결제 계산 방식만 다른 자식 클래스**이다.

즉, 상속의 핵심은 이것
> 공통 기능은 부모에게
> 
> 다른 기능은 자식에게

---
## 이 문제에서 반드시 알아야 하는 것!!
#### 1. private 필드는 자식도 직접 접근할 수 없다.
이거 중요함.
```
private int orderPrice;
```
이렇게 만들면 `VipMember`에서도 바로 `orderPrice`를 사용할 수 없다.

그래서 이렇게 하면 안 된다.
```
return orderPrice - discount; // 불가능
```
대신 getter를 사용해야 한다.
```
int price = getOrderPrice();
```
상속받았다고 해서 부모의 `private`필드까지 마음대로 만지는 것은 아니다.

---
#### 2. super( )는 부모 생성자를 호출한다.
```
super(name, orderPrice);
```
이 코드는 부모 클래스인 `Member`의 생성자를 호출한다.

비유하면:
```
VIP 회원 등록
↓

일단 기본 회원 등록부터 해야 함
↓

그 다음 VIP로 확장
```

---
#### 3. 오버라이딩은 "부모 기능을 자식 상황에 맞게 다시 작성" 하는 것
부모의 계산 방식:
```
return orderPrice;
```

VIP의 계산 방식:
```
return price - discount;
```
메서드 이름은 같지만, 자식 클래스에서 다르게 동작하게 만든다.

---
이 예제는 단순히 `extends`를 연습하는 문제가 아님.

진짜 목적은
> 공통 구조를 부모로 올리고, 달라지는 부분만 자식에서 바꾸는 감각을 익히는 것.

이 감각이 있어야 다형성에서
```
Member member = new VipMember("정윤", 30000);
```
같은 코드 볼 수 있게 됌.
