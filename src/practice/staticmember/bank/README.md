# 문제 : 은행 계좌 시스템
은행 계좌를 관리하는 `BankAccount` 클래스를 만든다.

각 계좌는 소유자와 잔액을 가진다.

모든 계좌가 공유하는 이자율은 `static` 변수로 관리한다.

입금 후 잔액이 100,000원 이상이면 `"VIP 고객"`을 출력한다.

---
## 문제 조건
- `BankAccount` 클래스를 만든다.
- 계좌마다 `owner`, `balance`를 가진다.
- 모든 계좌가 공유하는 `interstRate`를 static 변수로 만든다.
- `deposit()` 메서드로 입금한다.
- 입금 후 잔액이 100,000원 이상이면 `"VIP고객"`을 출력한다.
- 현재 잔액을 출력하는 메서드를 만든다.
---
## 코드 작성 전 생각하는 순서
```
계좌마다 주인은 다르다.
→ 인스턴스 변수

계좌마다 잔액도 다르다.
→ 인스턴스 변수

은행의 기본 이자율은 모든 계좌가 공유한다.
→ static 변수

입금 기능은 계좌 객체가 수행한다.
→ 인스턴스 메서드

잔액이 100000원 이상인지 판단한다.
→ 조건문
```

---
## 코드 작성하는 순서
```
1. BankAccount 클래스 생성

2. owner, balance 인스턴스 변수 선언

3. interestRate static 변수 선언

4. 생성자 작성

5. deposit 메서드 작성

6. 입금 후 if문으로 VIP 여부 판단

7. printInfo 메서드 작성

8. Main에서 계좌 객체 생성

9. deposit 호출

10. 계좌 정보 출력
```

---

```

```
---
## 핵심 원리
> 계좌 주인과 잔액은 객체마다 다르기 때문에 인스턴스 변수이고, 이자율은 모든 계좌가 공유하므로 static 변수이다.

---
- account1과 account2는 BankAccount 클래스에서 만들어진 실제 객체 (인스턴스)
- 쉽게 말해서, 
  BankAccount는 "계좌를 만드는 설계도"
  account1, account2는 그 설계도로 만들어진 "진짜 계좌"
