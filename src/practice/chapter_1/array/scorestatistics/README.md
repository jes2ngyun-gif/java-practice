### 시험 점수 통계 프로그램
### 문제 설명
정수 배열에 학생들의 시험 점수가 들어 있다.

이 배열을 이용해서 다음 값을 출력한다.
1. 전체 점수 출력
2. 총합
3. 평균
4. 최고 점수
5. 80점 이상인 학생 수

단, 이번 문제에서는 배열과 반복문만 사용.

---
### 이 문제에서 다루는 것

| 구분  | 내용                 |
| --- | ------------------ |
| 배열  | 여러 점수를 한 번에 저장     |
| 반복문 | 배열 전체 순회           |
| 조건문 | 80점 이상인지 판단        |
| 변수  | sum, max, count 관리 |
| 메서드 | 기능 분리              |
| 클래스 | 실행 클래스 구성          |

---
### 원리
```
배열 scores

index    0    1    2    3    4
값      70   85   90   60   100

반복문이 왼쪽부터 하나씩 확인한다.

70  → sum에 더함 / max와 비교 / 80점 이상인지 확인
85  → sum에 더함 / max와 비교 / 80점 이상인지 확인
90  → sum에 더함 / max와 비교 / 80점 이상인지 확인
60  → sum에 더함 / max와 비교 / 80점 이상인지 확인
100 → sum에 더함 / max와 비교 / 80점 이상인지 확인
```
배열 문제의 핵심은 거의 항상
>"칸을 하나씩 꺼내서 확인한다."

### 변수 변화 표
배열:
```
int[] scores = {70, 85, 90, 60, 100};
```

#### 총합 계산 흐름

| i | scores[i] |    sum 변화 |
| - | --------: | --------: |
| 0 |        70 |    0 → 70 |
| 1 |        85 |  70 → 155 |
| 2 |        90 | 155 → 245 |
| 3 |        60 | 245 → 305 |
| 4 |       100 | 305 → 405 |

결과:
```
sum = 405
```

---
#### 최고 점수 계산 흐름
처음:
```
max = scores[0]; // 70
```

| i | scores[i] | 비교         |   max 변화 |
| - | --------: | ---------- | -------: |
| 1 |        85 | 85 > 70    |  70 → 85 |
| 2 |        90 | 90 > 85    |  85 → 90 |
| 3 |        60 | 60 > 90 아님 |    90 유지 |
| 4 |       100 | 100 > 90   | 90 → 100 |

결과:
```
max = 100
```

### 핵심 원리
배열은 여러 값을 저장하는 고정된 칸이다.
```
scores[0]
scores[1]
scores[2]
...
```
처럼 인덱스로 접근한다.

반복문은 **배열의 인덱스를 하나씩 증가시키면서 모든 값을 확인한다.**
```
for (int i = 0; i < scores.length; i++)
```
이 문장은 배열 문제에서 **거의 기본 공식**이다.

---
### 알고 넘어가야 할 것.
#### 1. 배열의 인덱스는 0부터 시작한다.
```
scores[0]
```
이 첫 번째 값이다.

---
#### 2. 마지막 인덱스는 length -1이다.
배열 길이가 5라면 마지막 인덱스는 4다.
```
scores.length // 5
마지막 인덱스 // 4
```
그래서 반복문 조건은 보통 이렇게 쓴다.
```
i < scores.length
```

---
#### 3. 배열은 크기가 고정된다.
```
int[] scores = new int[5];
```
5칸짜리 배열인데 중간에 6칸으로 늘릴 수 없다.

**이 한계를 해결하려고 나중에 ArrayList를 쓴다.**

---
### 실무에서는
순수 배열보다 `List`를 더 많이 사용한다.

하지만 배열을 먼저 이해해야 하는 이유가 있다.

List, ArrayList, Stream도 결국 핵심은 같다.

```
여러 데이터가 있다
↓
하나씩 꺼낸다
↓
조건을 확인한다
↓
계산하거나 필터링한다
```
회원 목록, 주문 목록, 게시글 목록, 상품 목록 모두 이 원리로 처리한다.

### 다음 단계에서 이어질 내용

| 지금            | 다음          |
| ------------- | ----------- |
| 배열            | ArrayList   |
| scores[i]     | list.get(i) |
| scores.length | list.size() |

ArrayList를 사용해서 배열보다 왜 편한지 비교해본다.
