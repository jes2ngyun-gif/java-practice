# 게임 캐릭터(GameCharacter) 만들기
### 이거 왜 함?
게임을 생각해보자.

캐릭터의 체력(HP)은 굉장히 중요하다.

만약 외부에서
```
character.hp = -100;
character.hp = 999999;
```
같은 일이 가능하면 게임 벨런스가 망가진다.

그래서 게임 캐릭터는
- 체력을 숨기고(private)
- 공격받기
- 회복하기
- 현재 체력 확인하기

같은 행동을 통해서만 상태를 바꾼다.

---
## 문제 설명
게임 캐릭터를 만들어라.

캐릭터는 다음 정보를 가진다.
- 이름(name)
- 체력 (hp)

그리고 다음 기능을 제공한다.
- 데미지 받기
- 체력 회복하기
- 현재 체력 조회하기

## 핵심 원리
```

(private hp)

↓

(takeDamage, heal)

↓

객체가 자기 상태를 스스로 관리한다.

↓

잘못된 값이 들어오는 것을 막는다.
```

---
문제에서
```
this.hp = this.hp - damage;
```
가 나온다.

여기서

| 왼쪽          | 의미             |
| ----------- | -------------- |
| 왼쪽 this.hp  | 객체가 실제로 가진 체력  |
| 오른쪽 this.hp | 현재 체력          |
| damage      | 메서드가 전달받은 매개변수 |

예를 들어
```
character.takeDamage(30);
```

이면
```
현재 체력 : 100

↓

100 - 30

↓

70 저장
```
이 된다.
