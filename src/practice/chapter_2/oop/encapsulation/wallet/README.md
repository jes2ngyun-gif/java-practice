# 지갑 객체 만들기
## 문제 설명
지갑에는 돈이 들어 있다.

하지만 지갑 안의 돈을 외부에서 마음대로 바꾸면 안 된다.

그래서 `money` 필드는 `private`으로 숨기고, 돈을 넣고 빼는 행동은 메서드로만 처리한다.

```
Main
 ↓
Wallet 객체 사용
 ↓
deposit()  → 돈 넣기
withdraw() → 돈 빼기
getMoney() → 현재 돈 확인
```
---
## 문제의 코드 조건

| 조건      | 내용                     |
| ------- | ---------------------- |
| 클래스 이름  | `Wallet`               |
| 필드      | `private int money`    |
| 생성자     | 처음 가진 돈 설정             |
| 입금 메서드  | `deposit(int amount)`  |
| 출금 메서드  | `withdraw(int amount)` |
| 조회 메서드  | `getMoney()`           |
| 제한 조건 1 | 처음 가진 돈이 음수면 0원으로 설정   |
| 제한 조건 2 | 0원 이하 입금 금지            |
| 제한 조건 3 | 가진 돈보다 많이 출금 금지        |

---
## 원리 흐름

```
외부(Main)
   |
   | 직접 접근 X
   v
private int money

외부(Main)
   |
   | 허용된 메서드로만 접근
   v
deposit()
withdraw()
getMoney()
```
핵심은 이것이다.
```
돈(money)을 직접 만지지 못하게 막고
정해진 행동(deposit, withdraw)으로만 변경한다.
```

| 순서 | 생각                                 |

|  1 | 지갑이 기억해야 하는 값은 무엇인가? → 돈           |
|  2 | 돈은 외부에서 직접 바꾸면 안전한가? → 아니다         |

|  7 | 잘못된 입력을 막아야 한다 → 조건문 사용            |

## 실행 흐름
```
Main 실행

↓
Wallet wallet = new Wallet(10000);

↓
Wallet 객체 생성
money = 10000 저장

↓
wallet.deposit(5000);

↓
money = 15000

↓
wallet.withdraw(3000);

↓
money = 12000

↓
wallet.withdraw(20000);

↓
가진 돈보다 많으므로 출금 실패

↓
wallet.getMoney();

↓
현재 돈 12000원 출력
```

| 실행 코드               | money 값 | 설명               |
| ------------------- | ------: | ---------------- |
| `new Wallet(10000)` |   10000 | 처음 지갑에 10000원 있음 |
| `deposit(5000)`     |   15000 | 5000원 추가         |
| `withdraw(3000)`    |   12000 | 3000원 사용         |
| `withdraw(20000)`   |   12000 | 돈이 부족해서 실패       |

## 핵심 원리
### 캡슐화의 핵심
```
필드는 숨긴다.
private int money;

↓

외부에서는 직접 수정하지 못한다.

↓

메서드를 통해서만 변경한다.
deposit()
withdraw()

↓

객체가 자기 데이터를 스스로 지킨다.
```

---
### 왜 setter를 안 썻나?
입문자들이 자주 이렇게 만든다.
```
public void setMoney(int money) {
    this.money = money;
}
```
그런데 이 건 위 험 하 다 ! ! ! ! 
```
wallet.setMoney(-10000);
wallet.setMoney(99999999);
```
이렇게 외부에서 지갑 돈을 마음대로 바꿀 수 있다.

그래서 이 문제에서는 단순한  setter보다
```
deposit()
withdraw()
```
같은 **행동 중심 메서드**가 더 좋다.
