# 택배 배송 시스템
## 일반 배송과 새벽 배송 시스템 만들기

---
## 이 문제에서 사용하는 것

| 개념          | 사용 |
| ----------- | -- |
| 클래스         | O  |
| 객체          | O  |
| 생성자         | O  |
| private 필드  | O  |
| getter      | O  |
| if 조건문      | O  |
| 메서드         | O  |
| 상속(extends) | O  |
| super()     | O  |
| 오버라이딩       | O  |
| this        | O  |
| 출력 메서드      | O  |

---
## 문제 설명
쇼핑몰에는 두 가지 배송 방식이 있다.
#### 일반 배송
- 배송비 : 3000원

#### 새벽 배송
- 배송비 : 5000원
- 대신 "새벽 배송이 시작됩니다." 라는 추가 안내가 있다.

공통 정보는 부모 클래스가 담당하고,

새벽 배송만의 특징은 자식 클래스가 담당하도록 만들어라.

---
## 코드 작성 전에 생각하는 순서

### 1단계
배송마다 공통으로 필요한 것은?
```
상품 이름
가격
배송비 계산
주문 정보 출력
```
↓

부모 클래스

---
### 2단계
새벽 배송만 특별한 것은?
```
배송비가 다름
추가 안내 문구
```
↓

자식 클래스

---
### 3단계
배송비 계산 방식만 다르네??'
→ 오버라이딩 사용

---
## 코드를 작성하는 순서
```
1. Delivery 부모 클래스 생성

2. 필드 만들기
- productName
- price

3. 생성자 작성

4. 배송비 계산 메서드 작성

5. 주문 정보 출력 메서드 작성

6. DawnDelivery 자식 클래스 생성

7. super() 호출

8. 배송비 계산 메서드 재정의

9. 추가 안내 메서드 작성

10. Main에서 객체 생성 후 실행
```
---
## 클래스 관계
```
                Delivery
          (부모 클래스)

      상품명
      가격
      배송비 계산()
      주문정보 출력()

                  ↑

           extends

                  ↑

             DawnDelivery
          (자식 클래스)

       배송비 계산() 재정의
       새벽배송안내()
```
---
## 실행 흐름
```
Main
 ↓

new Delivery()
 ↓

Delivery 생성자

new DawnDelivery()
 ↓

DawnDelivery 생성자

super()
 ↓

Delivery 생성자

printOrderInfo()

↓
calculateShippingFee()

↓
부모 또는 자식 메서드 실행
(오버라이딩)
```
---
## 이 문제의 핵심 원리
상속은
```
복붙하기 위해 사용하는 기술이 아니다.
```
진짜 목적은:
> 공통 부분은 부모에게
> 
> 달라지는 부분은 자식에게

를 실현하는 것이다.

---
### 부모 클래스 안에서
```
calculateTotalPrice()
```
얘가
```
calculateShippingFee()
```
얘를 호출하고 있다.

```
return price + calculateShippingFee();
```
이게 굉장히 중요하다.

왜냐하면

객체가 새벽 배송 객체라면

부모 클래스 안에 있더라도

자식이 재정의한 메서드가 실행된다.

이 현상이 바로 다음 강의인 **다형성**의 시작점이다!!!!!!!1
